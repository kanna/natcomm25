\item {\it Methodology}\label{sec:r1-meth}


  \action: We predicted uncertainty by computing the pointwise
  standard deviation from an ensemble of three-dimensional temperature
  models predicted with stochastic sequential simulation. Stochastic
  sequential simulation predicts multiple models (i.e., geostatistical
  realizations) given a set of observed data and a spatial covariance
  matrix, as revealed by a variogram model.  In stochastic sequential
  simulation, the simulation grid is visited following a random
  path. At each location along these random paths, the local kriging
  estimate and kriging variance are computed using a spatial
  covariance matrix and existing direct observations, as well as, if
  available, previously visited values. The kriging variance and
  estimate are used to sample from the probability distribution of the
  property of interest as estimated from existing data. Each time the
  simulation runs, a new random path is defined and different results
  are produced. Each realization matches the observed data, the
  primary and secondary statistics as revealed from the observed data
  and the imposed spatial continuity pattern.  The set of
  geostatistical realizations represents the posterior distribution of
  the phenomena one is trying to model, and this ensemble can be used
  to estimate uncertainty by computing local statistics at the grid
  cell level. A complete description of this methodology is provided
  in \cite{duarte2025}.

  We have modified the text to read:

\begin{quote}

  Our methodology relies on ensembles of geostatistical realizations,
  each representing a state of an ocean variable field conditioned
  \emph{a priori} by the deterministic CMEMS NEMO model
  \cite{madec97,madec15,CMEMS_IBI_MFC} using the Iberian Biscay Irish
  domain. In doing so we use a variogram model describing the temporal
  dependency between days over a 14 day period. Realizations are
  generated via stochastic sequential simulation: the grid is visited
  along a random path and, at each node, a local kriging mean and

  available conditioning data, from which a value is drawn from the

\end{quote}

\item \reply: L231: The text mentions that the algorithm ensures
  “vehicle endurance and safety constraints.” Please describe how you
  constrain the algorithm for safety.

  \action: We enforce both endurance and safety constraints in two
  complementary ways. First, we constrain endurance by limiting each
  vehicle’s planned route to an effective maximum distance derived
  from its mission time budget, while accounting for mandatory
  surfacing time for communications and geolocation. We constrain
  navigational safety by planning only within an admissible-area map
  where unsafe regions are masked out (e.g., land/shoreline,
  shallow-water areas below a conservative depth threshold, and
  locally known/identified hazards such as high-traffic zones, buoys,
  or fishing gear). Every candidate path is then algorithmically
  constrained. These constraints are implemented as hard feasibility
  checks, so the route planner only returns routes that satisfy both
  endurance and safety requirements. This is illustrated by the
  boundaries in the uncertainty field shown in the zoomed area of the
  new Fig. 4. More details on how this is mathematically incorporated
  in the cost function are provided in \cite{bernacchi2025} and
  \cite{bernacchi26}. We have modified the text to read:

       \ldots while enforcing an effective travel budget (endurance)
       and an obstacle-masked admissible area (safety)
       \cite{bernacchi2025}. This is illustrated by the boundaries in

    \end{quote}

\item {\it General Comments:}\label{sec:r2-genl}

    \reply: The paper is generally well constructed, well written and
    accurate. I appreciate the authors’ efforts. For real-time ocean
    awareness, glider missions are part of a sample-assimilate
    predict-direct loop. Here, the authors review an experiment where
    they demonstrate an ability to send gliders on sampling missions,
    assimilate the data, predict the next day with a low computational
    cost forecast model, and then generate guidance based on the
    forecast. All of these components have been developed by the authors
    and their collaborators. Bring all these components together
    represents a significant accomplishment. Some parts of their work
    are novel - the forecast model has low computational cost, and thus
    appropriate for low-powered/limited-resource environments. The
    forecast model can also generate uncertainty maps, which are useful
    for generating glider guidance. The gliders are of their own design.
    I also appreciate the discussion of the observed subgridscale
    variability in section 5. Some parts of the paper would benefit
    additional attention. There is no discussion of the gliders speed
    through water nor typical water velocities in the region. There is
    no comparison of predicted glider trajectories versus guidance
    fields. Also, there is no comparison of predicted trajectories with
    actual glider trajectories. The combination of these things makes it
    hard for me to evaluate how well the guidance was followed. There is
    no comparison of the low-cost forecast model versus the global
    forecast that the low-cost model is based on. So, it makes it hard
    to say if this approach improves forecast skill over the global
    model forecast. I think the paper would be strengthed by showing how
    well the gliders followed their guidance, showing that the
    observations reduce uncertainty (maybe just put the glider
    trajectories on the uncertainty maps?), and showing the performance

  \reply: Lines 410-439: An alternative explanation for assimilation
